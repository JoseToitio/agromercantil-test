Ajuda para Validação FIPE

Atenção: Para que a validação FIPE funcione corretamente, é importante seguir as instruções abaixo.
Como funciona a validação FIPE?

A validação FIPE é um processo que verifica se os dados do caminhão (marca, modelo e ano) existem na tabela FIPE e obtém o valor de referência. Além dos parâmetros obrigatórios, a API também aceita parâmetros opcionais como mês de referência e token de assinatura.

Parâmetros da API FIPE:
  • Obrigatórios:
    • brand - Marca do veículo
    • model - Modelo do veículo
    • year - Ano de fabricação
  • Opcionais:
    • reference - Mês e ano de referência (padrão: 278)
    • token - Token de assinatura para acesso à API FIPE
Problemas comuns:
  1. Erro 400 (Bad Request):
    • Ocorre quando os dados enviados não são encontrados na tabela FIPE.
    • Verifique se a marca, modelo e ano estão escritos exatamente como aparecem na tabela FIPE.
  2. Dados não encontrados:
    • Alguns modelos mais antigos ou específicos podem não estar disponíveis na tabela FIPE.
    • Tente usar a página de teste FIPE para verificar quais dados estão disponíveis.
Dicas para uso correto:
  1. Sempre selecione a marca primeiro, depois o modelo e por último o ano.
  2. Aguarde o carregamento completo das opções antes de prosseguir para o próximo campo.
  3. Se receber um erro, verifique se os dados estão corretos usando a página de teste FIPE.
  4. Lembre-se que a tabela FIPE é atualizada periodicamente, então alguns valores podem mudar com o tempo.
Solução de problemas:

Se continuar enfrentando problemas com a validação FIPE, você pode:

  • Usar a página de Teste FIPE para verificar quais dados estão disponíveis.
  • Verificar o console do navegador para ver mensagens de erro detalhadas (F12 > Console).
  • Tentar com diferentes combinações de marca, modelo e ano.
